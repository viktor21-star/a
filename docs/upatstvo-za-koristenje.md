# Упатство За Користење

Ова упатство е за секојдневна работа со системот `Печење`.

Системот има 2 главни типа на корисници:

- `Администратор`
- `Оператор`

## Најава

1. Отвори ја web апликацијата или Android апликацијата.
2. Внеси `корисничко име` и `лозинка`.
3. Ако корисникот е активен и лозинката е точна, системот ќе те најави.

Ако корисникот е деактивиран:

- нема да може да се најави

Ако апликацијата не е на последна верзија:

- ќе побара задолжителен update
- старата верзија нема да дозволи понатамошна работа

## Упатство За Администратор

Администраторот има почетен екран со големи кочки за модулите.

### 1. План за печење

Во `План за печење` прво се избира:

- `Пекара`
- `Печењара`

Потоа се внесува:

- `локација`
- `време на печење`
- `количина`

Во овој дел не се внесува артикал. Планот е по:

- локација
- време
- количина
- модул (`Пекара` или `Печењара`)

Овој план му се прикажува на операторот за неговата локација.

### 2. Реално печење

Во `Реално печење` администраторот има преглед на внесовите што ги прават операторите.

Се гледа:

- кој оператор внел
- за која локација
- кој модул
- кои артикли
- колкава количина
- слика
- време на внес

### 3. Аларми

Во `Аларми` се гледаат отстапувањата меѓу:

- `план`
- `реално печење`

Овој дел служи за брз увид каде има проблем.

### 4. Извештаи

Во `Извештаи` администраторот гледа:

- план vs реализација
- испечено
- `Пијара / Класа Б`

Тука има и export опции.

### 5. Локации

Во `Локации` има само преглед.

Се гледа:

- шифра на локација
- име на локација
- регион
- печки за `Пекара`
- печки за `Печењара`

Нема рачен внес на локации, затоа што локациите доаѓаат од source систем.

### 6. Шифарници

Во `Шифарници` има:

- `Артикли`
- `Локации`
- `Печки`
- `Термини`
- `Причини`

`Артикли` и `Локации` се само преглед.

`Печки`, `Термини` и `Причини` се работни делови што може да се менуваат.

### 7. Печки

Во `Печки` се внесува по локација:

- тип на печка за `Пекара`
- број на печки за `Пекара`
- капацитет по печка за `Пекара`
- тип на печка за `Печењара`
- број на печки за `Печењара`
- капацитет по печка за `Печењара`

Пример:

- `Пекара`: капацитет `48`
- `Печењара`: капацитет `12`

### 8. Термини

Во `Термини` администраторот може:

- да додаде нов термин
- да пребарува
- да активира / деактивира
- да избрише термин

Термините се користат во планирањето.

### 9. Причини

Во `Причини` администраторот може:

- да додаде причина
- да пребарува
- да активира / деактивира
- да избрише причина

Категории:

- `разлика`
- `отпад`
- `доцнење`

### 10. Корисници

Во `Корисници` има 2 секции:

- `Нов корисник`
- `Постоечки корисници`

#### Нов корисник

При отворање нов корисник се внесува:

- работна локација
- име и презиме
- корисничко име
- лозинка
- улога
- печка за `Пекара`
- печка за `Печењара`
- привилегии за:
  - `Пекара`
  - `Печењара`
  - `Пијара`

#### Постоечки корисници

Во оваа секција може:

- да се пребаруваат корисници
- да се менуваат привилегии
- да се активира / деактивира корисник
- да се смени лозинка

Ако операторот нема привилегија за модул:

- тој модул нема да му се појави на почетниот екран

### 11. Верзија

Во `Верзија` се одржува:

- последна верзија
- build број
- датум на release
- download линк
- дали update е задолжителен

## Упатство За Оператор

Операторот гледа само модули што му се дозволени за неговата локација.

Може да му се појават:

- `Пекара`
- `Печењара`
- `Пијара`

Ако нема дозвола за модул:

- модулот не се прикажува

### 1. Пекара

Во `Пекара` операторот:

1. слика печење
2. избира артикал
3. внесува количина
4. снима внес

Може да додаде повеќе артикли во еден внес.

Операторот исто гледа:

- план за печење за неговата локација
- време
- количина

### 2. Печењара

Во `Печењара` процесот е ист:

1. слика
2. избор на артикал
3. количина
4. снимање

И тука може повеќе артикли во еден внес.

### 3. Пијара

Во `Пијара` важи:

- една пријава = една слика
- се избира артикал
- се внесува `вкупна количина`
- `Класа Б` е вклучено по default
- ако е штиклирано `Класа Б`, се внесува и `количина за Класа Б`

Поле `шифра` значи:

- шифра на артикал
- не е количина

### 4. Успешен и неуспешен внес

Кога внесот е успешен:

- има звучен сигнал
- има вибрација ако телефонот поддржува
- операторот се враќа на почетниот екран

Кога нема интернет:

- внесот се снима локално
- кога ќе се врати интернет, автоматски се праќа до серверот

Кога внесот е неуспешен:

- операторот добива порака да провери:
  - интернет
  - дали серверот работи
  - ако проблемот остане, да контактира администратор

## Кои програми и технологии се користени

Системот е направен со:

- `React`
- `TypeScript`
- `Vite`
- `Capacitor Android`
- `Gradle`
- `ASP.NET Core 8`
- `Dapper`
- `SQLite`
- `SQL Server` за source податоци
- `GitHub Actions` за Android build

### Што точно користи системот

- `frontend`
  - React web интерфејс
  - истиот frontend оди и во Android APK

- `backend`
  - ASP.NET Core API
  - локално сервира web + API + APK download

- `SQLite`
  - локална operational база на овој компјутер
  - тука се чуваат:
    - планови
    - операторски внесови
    - печки
    - термини
    - причини
    - корисници

- `SQL Server source база`
  - од таму се читаат:
    - локации
    - артикли

- `APK`
  - Android апликација за оператори и администратори

## Каде се стартува системот

На овој компјутер системот се стартува преку:

- [Pecenje Start.desktop](/home/zitomarketi/Desktop/Pecenje%20Start.desktop)
- [Pecenje Stop.desktop](/home/zitomarketi/Desktop/Pecenje%20Stop.desktop)

Тие го стартуваат:

- backend
- web
- локален API
- APK download path

## Што е важно да се знае

- `Артикли` и `Локации` не се внесуваат рачно
- operator модулите зависат од привилегиите по локација
- постара верзија на апликацијата не смее да продолжи ако има понова задолжителна верзија
- ако нема интернет, внесовите не се губат
